Chapter 03: Hoisting in JavaScript

1. What is Hoisting?

Common explanation:
Hoisting in JavaScript is a behavior where variable and function declarations are moved to the top of their scope before the code is executed.

‚ùå This explanation is misleading ‚Äî nothing is physically moved.

2. Real Definition of Hoisting

Hoisting:
JavaScript knows about variables and functions before execution starts.

üëâ JavaScript scans the code before execution and remembers declarations.

3. Hoisting Behavior Summary

  • Function declarations are hoisted and initialized
  • var is hoisted and set to undefined
  • let and const are hoisted but uninitialized

4. Example 1 ‚Äî Hoisting with let

console.log(b);
let b = 5;
console.log(b);
‚ùó Accessing let before declaration causes ReferenceError (TDZ).

7. Example 3 ‚Äî Function Expression Hoisting

Case 1: Call Before Function Expression

sayHi(); // Error
// console.log(sayHi); // undefined

var sayHi = function () {
  console.log("Hi!");
};

Case 2: Call After Function Expression

var sayHello = function () {
  console.log("Hello!");
};

sayHello();

Explanation

  • Function expression behavior depends on how it is declared
  • var ‚Üí hoisted and initialized with undefined
  • let / const ‚Üí hoisted but uninitialized (TDZ)
‚ùó Calling a function expression before declaration:
  • var ‚Üí TypeError (not a function)
  • let / const ‚Üí ReferenceError (TDZ)
