What is Node.js?

Node.js is a JavaScript runtime environment that allows us to run JavaScript outside the browser. It is commonly used for backend development and building server-side applications.

Steps to Install Node.js

1) Open your browser and go to the official Node.js website.

2) Download the LTS (Long Term Support) version.

3) Run the installer.

4) Click "Next" and follow the installation steps.

5) After installation, Node.js and npm (Node Package Manager)
   will be installed automatically.

Check Installation

# check node version
node -v
# check npm version
npm -v

What We Get with Node.js

Definition:

When we install Node.js on our system, several important tools are installed automatically. These tools allow developers to run JavaScript outside the browser and manage packages required for building applications.

Tools Installed with Node.js:


1) Node.js Runtime
   • Allows us to run JavaScript outside the browser.

2) npm (Node Package Manager)
   • Used to install external libraries and packages.

3) npx
   • Used to run Node packages without installing them globally.

4) Node REPL
   • Interactive environment where we can run JavaScript program in chunks
     directly in the terminal (similar to browser console).

Example Commands:

node

Executing First JavaScript Program in Node.js

Definition:

After installing Node.js, we can run JavaScript programs directly from the terminal. To execute a JavaScript file, we use the node command followed by the file name.

Example 1: Returning Greeting Message

// greet1.js
function greet(name)
{
    let greet_msg = `hello ${name}`
    return greet_msg
}
let greeting = greet("md shahid")
console.log(greeting)

// command to run
$ node greet1.js

Example 2: Printing Greeting Inside Function

// greet2.js
function greet(name)
{
    let greet_msg = `hello ${name}`
    console.log(greet_msg)
}
greet("md shahid")

// command to run
$ node greet2.js

Example 3: Using Callback Function

// greet3.js

function create_greet(name, callback)
{
    let greet_msg = `hello ${name}`
    callback(greet_msg)
}

create_greet("md shahid", (msg) => {
    console.log(msg)
})

// command to run
$ node greet3.js

Creating a Server Using HTTP Module

What is HTTP Module?

The HTTP module is a built-in module in Node.js(core module) that allows us to create web servers. It helps the server communicate with clients (like browsers) using the HTTP protocol. Using the HTTP module we can: • Create a server • Handle client requests • Send responses back to the browser

Example: Creating a Server and Understanding Request Object

// example: creating a server and understanding request object

// importing http module
const http = require('http')

// creating a server
const server = http.createServer((req, res) => {

  console.log("req:", req)
  console.log(req.url)
  console.log(req.method)

})

const Port = 3000

// listening a request
server.listen(Port, () => {
  console.log(`server is running at http://localhost:${Port}`)
})

// by default browser sends a GET request and url "/"

// command to run
$ node server.js

Example: Sending a Response to the Browser

// sending response

const http = require("http")

const server = http.createServer((req, res) => {

  console.log("new request")

  res.setHeader('content-type', "text/html")

  res.write(`
      <html>
         <header>
           <title>response</title>
         </header>
         <body>
            hello subscribe me
         </body> 
      </html>
  `)

  res.end()

})

const port = 3001

server.listen(port, () => {
  console.log(`server is running at http://localhost:${port}`)
})

// problem here
// whatever the url of request we always send the same response
// this is not a good design
// we should create different endpoints for different request URLs

// command to run
$ node server.js

Example: Sending Response Based on Requested URL (Routing Requests)

Routing means sending different responses based on the requested URL. In Node.js we can check req.url and return different HTML pages for different endpoints. 
// sending a response based on requested URL (routing requests)

const http = require("http")

const server = http.createServer((req, res) => {

  if(req.url == "/")
  {
    res.setHeader('content-type', "text/html")

    res.write(`
      <html>
        <header>
          <title>response</title>
        </header>
        <body>
          this is a home page
        </body>
      </html>
    `)

    return res.end()
  }

  else if(req.url == "/product")
  {
    res.setHeader('content-type', "text/html")

    res.write(`
      <html>
        <header>
          <title>response</title>
        </header>
        <body>
          here is your product
        </body>
      </html>
    `)

    return res.end()
  }

  res.setHeader('content-type', "text/html")

  res.write(`
      <html>
        <header>
          <title>response</title>
        </header>
        <body>
          for any request other than / and /product
        </body>
      </html>
  `)

  return res.end()

})

const port = 3000

server.listen(port, () => {
  console.log(`server is running at http://localhost:${port}`)
})

// command to run
$ node server.js

Node REPL (Read–Eval–Print Loop)

Definition

Node REPL stands for Read – Eval – Print – Loop. It is an interactive shell provided by Node.js where we can execute JavaScript code line by line. It works similar to the browser console. Developers can quickly test JavaScript code without creating a separate JavaScript file.

How to Start Node REPL

// start Node REPL
$ node

Example: Executing JavaScript Code in REPL

// simple calculation
> 5 + 10
15

// declaring variable
> let name = "shahid"

// printing variable
> name
"shahid"

// defining a function
> function greet(name){
... return `hello ${name}`
... }

// calling the function
> greet("shahid")
'hello shahid'
The REPL reads the JavaScript command, evaluates it, prints the result, and then waits for the next command. This cycle continues until we exit the REPL.
