Chunks & Buffer

What is Chunk?

When a client sends data (like in a POST request), Node.js does NOT receive the whole request body at once.

that is If the client sends a large file (e.g.,5 MB file), Node does not load all 5 MB into memory at once. Instead, it receive the data in small pieces called chunk

Client
↓
Request Body
↓
Chunk 1
Chunk 2
Chunk 3
Chunk 4
...
Chunk N

1️⃣ What does a chunk look like?

<Buffer 6e 61 6d 65 3d 73 68 61 68 69 64>

2️⃣ When do you get only one chunk?

If request body is small, Node.js may receive only one chunk.

What is Buffer?

A Buffer in Node.js is a special object used to store incoming data in binary format.

Network data (HTTP request) comes in raw binary, not plain text.

3. Flow Explanation

1. Client sends form data
2. Node receives data in chunks
3. "data" event collects chunks
4. "end" event means all chunks received
5. Buffer.concat() joins chunks
6. Convert to string using toString()
7. Send response to client

step2. joining each chunks

// reading and joining chunks
const http = require("http");

const server = http.createServer((req,res)=>{
  if(req.url === "/"){
    res.write(` <h1>Home Page</h1> <form action="/submit-details" method="post"> <input type="text" name="userName"> <button>Submit</button> </form> `);
    res.end();
  }

  else if(req.url === "/submit-details" && req.method === "POST"){
    let body = [];

    // collect chunks
    req.on("data", (chunk)=>{
      console.log("chunk:", chunk);
      body.push(chunk);
    });

    // join chunks
    req.on("end", ()=>{
      const finalBuffer = Buffer.concat(body);
      //convert buffer to string
      constfinalData = finalBuffer.toString();

      console.log("Final Data:", finalData);

      res.end();
    });
  }
});

server.listen(3000, ()=>{
  console.log("Server is listening on port 3000");
});

How Chunk Joining Works

1. Data comes in multiple chunks
2. Each chunk is stored in an array
3. Buffer.concat(body) joins all chunks
4. .toString() converts binary data into readable text
5. Final data becomes usable (e.g. userName=shahid)
